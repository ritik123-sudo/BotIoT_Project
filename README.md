# BotIoT — Interactive Threat Detection Tool

A machine-learning-powered web tool to detect IoT botnet attacks using the **BotIoT dataset** from Kaggle.

---

## Quick Start

### 1 · Install dependencies
```bash
pip install -r requirements.txt
```

### 2 · Train the model
```bash
python train.py
```
This reads `dataset/bot_iot.csv`, trains a **Random Forest** classifier, evaluates it, and saves the model to `model/model.pkl`.  Expected accuracy: **~99%** (BotIoT is heavily imbalanced with 73 % attack traffic).

### 3 · Launch the web app
```bash
python app.py
```
Open **http://127.0.0.1:5000** in your browser.

## Features

| Feature | Description |
|---|---|
| **Manual Input** | Fill form fields for any packet and get an instant prediction |
| **Load Sample** | Paste a raw CSV row from the dataset for quick testing |
| **Random Sample** | Auto-fill the form with random plausible values |
| **Batch Upload** | Drop a CSV file — up to 500 rows analysed at once |
| **Export Results** | Download batch predictions as a CSV |
| **Detection History** | Running log of all predictions in the session |

---

## API Endpoints

| Method | URL | Body | Description |
|---|---|---|---|
| GET | `/` | — | Web interface |
| POST | `/predict` | JSON object with feature keys | Single prediction |
| POST | `/predict_raw` | `{"row": "<csv string>"}` | Predict from raw CSV row |
| POST | `/predict_batch` | `{"csv": "<csv text>"}` | Batch predictions |

### Example `/predict` request
```json
POST /predict
{
  "proto": "tcp",
  "pkts": 10,
  "bytes": 680,
  "state": "CON",
  "dur": 1453.9,
  "rate": 0.00619
}
```

### Example response
```json
{
  "prediction": 1,
  "attack_probability": 0.97,
  "confidence": 0.97
}
```

## Model Details

- **Algorithm**: Random Forest (100 trees, max depth 20)
- **Target**: `attack` column — binary (0 = Normal, 1 = Attack)
- **Features**: all numeric + encoded categorical columns after dropping identifiers
- **Class balancing**: `class_weight="balanced"` to handle skewed distribution
- **Evaluation**: Accuracy + ROC-AUC on 20 % held-out test split

---

## Dataset

Download `bot_iot.csv` from [Kaggle](https://www.kaggle.com/) and place it in `dataset/`.  
The dataset should have these 35 columns (some are dropped during training):

`pkSeqID, stime, flgs, proto, saddr, sport, daddr, dport, pkts, bytes, state, ltime, seq, dur, mean, stddev, smac, dmac, sum, min, max, soui, doui, sco, dco, spkts, dpkts, sbytes, dbytes, rate, srate, drate, attack, category, subcategory`
